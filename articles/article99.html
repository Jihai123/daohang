分析：GPT-4o 与 GPT-4 Turbo
共同作者
阿卡什·夏尔马
西德·西塞帕利
安妮塔·科科夫斯卡

本周，OpenAI 发布了 GPT-4o，这是一种多模式模型，与最新的 GPT-4 Turbo 版本相比，速度提高了 2 倍，成本降低了 50%，速率限制提高了 5 倍。

它具有令人印象深刻的多模式能力；与这个模特聊天是如此自然，你可能会忘记它是人工智能（就像她一样）。

和许多人一样，我们很高兴看到这些进步，因此我们进行了一项实验，在三个任务上比较 GPT-4o 和它的前身：分类、数据提取和语言推理。

我们了解到 GPT4o 在所有三项任务上都优于 GPT-4 Turbo，但是：

  • 对于复杂的数据提取任务，准确性是关键，这两种模型仍然达不到标准。
  • 对于客票分类，GPT4o 与 GPT4-Turbo 相比具有最佳精度。与 Claude 3 Opus 和 GPT-4 相比，它仍然具有最好的精度。
  • 在推理方面，GPT-4o 在日历计算、时间和角度计算、反义词识别等任务上进行了改进。然而，它在文字处理、模式识别、类比推理和空间推理方面仍然存在困难。

请阅读以下部分的完整分析。

方法

本次分析的主要重点是分析 GPT-4o 相对于最新 GPT-4 Turbo 模型的改进 ( gpt-4-0613)。

我们查看标准基准、社区运行的数据，并进行一系列我们自己的小规模实验。

在接下来的两节中，我们将介绍：

  • 性能对比 （L‍‍a‍‍t‍‍e‍‍n‍‍c‍‍y‍‍、‍‍ ‍‍T‍‍h‍‍r‍‍o‍‍u‍‍g‍‍h‍‍p‍‍u‍‍t‍‍ ）
  • 标准基准比较 （例如：GPT-4o 与 GPT-4 之间数学任务的报告性能如何？）

然后，我们进行小型实验并比较三个任务的模型：

  • 数据提取
  • 分类
  • 语文推理

吞吐量比较

在吞吐量方面，之前的 GPT 模型是落后的；最新的 GPT-4 Turbo 每秒仅生成 20 个代币。然而，GPT-4o 已经做出了重大改进，每秒可以产生 109 个代币。

即使有了这样的改进，GPT4o 仍然不是最快的模型（Groq 上托管的 Llama 每秒生成 280 个代币），但其先进的功能和推理使其成为实时 AI 功能的不错选择。

报告的能力

标准基准

当新模型发布时，我们从技术报告中报告的基准数据中了解它们的功能。下图将 GPT-4o 在标准基准测试中的性能与排名前五的专有模型和一个开源模型进行了比较。

该图表的主要要点：

  • 在推理能力基准 MMLU 上，GPT-4o 得分为 88.7%，比 GPT-4 Turbo 提高了 2.2%。推理仍然是所有 GPT 模型的标志性功能，始终使它们与众不同。我们将在下一节中测试这些改进的影响。
  • GPT-4o 在 GPQA（生物学、物理和化学）、数学和 HumanEvals （编码）方面显示出显着改进。
  • 在多语言小学数学基准 MGSM 上，GPT-4o 显示出与最高能力模型 Claude 3 Opus 类似的功能。
  • 有趣的是，在需要复杂推理和算术的 DROP 数据集上，GPT-4 Turbo 的性能优于较新的 GPT-4o，尽管 GPT-4o 是该模型的增强版本。

Elo 排行榜

在 GPT-4o 发布之前，OpenAI 团队“秘密”在 LMSYS Chatbot Arena 中添加了该模型，因为im-also-a-good-gpt2-chatbot.该平台允许您提示两个匿名语言模型，投票选出最佳响应，然后透露他们的身份。

GPT-4o 目前是该排行榜中最先进的模型，获得了令人印象深刻的 1310 ELO 排名，与前 5 名性能模型相比有显着提升。

基准和众包评估很重要，但它们并不能说明全部情况。要真正了解您的人工智能系统的性能，您必须深入研究并针对您的用例评估这些模型。

现在，让我们在可能对您的项目有用的三个任务上比较这些模型。

任务1：数据提取

对于此任务，我们将比较 GPT-4 Turbo 和 GPT-4o 从合约中提取关键信息的能力。我们的数据集包括公司与其客户之间的主服务协议 (MSA)。合同的长度各不相同，有的短至 5 页，有的长于 50 页。

在本次评估中，我们将提取合同标题、客户名称、供应商名称、终止条款详细信息、是否存在不可抗力等总共 12 个字段。

您可以检查我们的原始提示和我们期望模型返回的 JSON 模式：

您是一名合同审核员，致力于帮助审核并购交易后的合同。您的目标是分析提供的文本并返回关键数据点，重点关注合同条款、风险和其他重要特征。您应该仅使用提供的文本来返回数据。

根据提供的文本，使用架构创建有效的 JSON：
{
Contract_title: string, // 协议名称

vendor: string, // 这是提供服务的供应商
effective_date: 日期, // 格式为 m/d/yyyy
initial_term: string, // 协议的长度（例如 1 年、5 年、18 个月等）
extension_renewal_options: string, // 合同中是否有延期或续订选项？
auto_renewal: string, // 该协议是否设置为自动续订？

force_majeure_pandemic: string, // 不可抗力是否包括病毒爆发、流行病或流行病事件？

}

我们收集了 10 份合约的真实数据，并使用Vellum 评估来设置 12 个自定义指标。这些指标将我们的真实数据与模型生成的 JSON 中每个参数的 LLM 输出进行比较。然后，我们对 GPT-4 Turbo 和 GPT-4 进行测试，以下是我们评估报告的结果：

我们从合约中提取了12条信息：

  • GPT-4o 在 12 个字段中的 6 个字段中表现优于 GPT-4 Turbo，在 5 个字段中保持相同的结果，并在 1 个字段中表现出性能下降。
  • 从绝对角度来看，GPT-4和GPT-4o在大多数领域都只正确识别了60-80%的数据。对于精度很重要的复杂数据提取任务，这两种模型都达不到标准。可能有一种方法可以通过先进的提示技术（例如少击或思维链提示）获得更好的结果。
  • 正如预期的那样，GPT-4o 在 TTFT（第一个令牌的时间）上比 GPT-4 Turbo 快 50-80%，这进一步有助于 GPT-4o 在这一头对头比较中。

获胜者：GPT-4o，与 GPT-4 Turbo 相比，其质量和延迟更好。然而，这可能仍然不是最适合这项工作的模型。需要进一步评估和及时测试来确定最佳选择。

任务2：分类

在此评估中，我们让 GPT-4o 和 GPT-4 确定客户支持票证是否已解决。在我们的提示中，我们提供了有关客户票证何时关闭的明确说明，并添加了一些示例来帮助解决最困难的情况。

我们运行评估来测试模型的输出是否与 100 个标记测试用例的真实数据相匹配，结果如下：

要点：

  • 与 GPT-4 Turbo 相比，GPT-4o 性能提升了 7%。有趣的是，在我们去年 12 月使用相同数据进行的评估中，GPT-4 Turbo 得分为 65%，高于今天的结果。我们肯定需要进一步分析这一点，但人们可以假设 GPT-4 Turbo 会随着时间的推移而退化。
  • 另一方面，GPT-4 达到了令人印象深刻的 78% 准确率，Claude 3 Opus 达到了 72%，两者都明显优于 GPT-4o。

准确性很重要，但不是唯一需要考虑的指标，特别是在误报（错误地将未解决的票证标记为已解决）可能导致客户不满意的情况下。然后我们计算了这两个模型的精确率、召回率和 f1 分数，同时还添加了 Claude 3 Opus 和 GPT-4 以进行良好的衡量：

要点：

  • GPT4o：GPT4o 具有最高的精度 (88.00%)，是避免误报的最佳选择，确保已解决的工单确实得到解决。
  • GPT-4 Turbo 和 Claude 3 Opus：这两种型号也都具有高精度（83.33%），使其成为很好的替代品。然而，与 GPT4o 相比，它们的召回率较低。
  • GPT-4：尽管具有最高的召回率（87.50%）和良好的 F1 分数（81.67%），但其精确度（76.56%）低于其他模型。这可能会导致更多的误报，在这种情况下这是不太理想的。

获胜者：GTP-4o 表现出比所有其他型号更好的精度。这将是我们执行此任务的首选模型。同样重要的是要强调，决定使用哪个模型取决于您的任务以及您想要在准确性、精确度和召回率之间取得的平衡。还值得一提的是，您可以使用CoT等高级提示技术来改进特定用例的模型结果。

‍

任务 3：推理

虽然 GPT-4 Turbo 在许多推理任务中表现出色，但我们之前的评估表明它在处理语言推理问题上表现不佳。据 OpenAI 称，与 GPT-4 Turbo 相比，GPT-4o 在推理任务方面表现出显着改进。

GPT-4o真的更好吗？

为了看看新模型是否更好，我们选择了一组 16 个言语推理问题作为测试的基石。

这是一个谜语示例及其来源：

💡言语推理问题：

1. 选择最能完成这个类比的词：羽毛之于鸟，正如鳞片之于_______。

A) 爬行动物

B) 狗

C) 鱼

D) 植物

答案：爬行动物

下面是我们在 Vellum 提示环境中运行的初始测试的屏幕截图：

现在，让我们对所有 16 个推理问题进行评估。

在下图中，您可以看到 GPT-4o 表现出了比其前身更好的推理能力，达到了 69% 的准确率，而 GPT-4 Turbo 的准确率为 50%。

从我们给模型提供的示例中，我们可以看到 GPT-4o 在以下推理任务中变得更好：

  • 日历计算：准确识别特定日期重复的时间。
  • 时间和角度计算：精确计算时钟上的角度。
  • 词汇（反义词识别）：有效识别反义词并理解词义。

而且，它仍然难以完成以下推理任务：

  • 单词操作：字母更改后难以识别和生成有意义的单词。
  • 模式识别：难以识别和应用复杂的重排模式。
  • 类比推理：准确理解和匹配关系类比的问题。
  • 空间推理：可视化空间运动和计算距离的问题。

获胜者： GPT4o 它肯定更好，但在某些推理任务上仍然遇到困难。

‍

概括

在本文中，我们研究了标准基准，进行了小规模实验并研究了独立评估。以下是我们的调查结果摘要。

概括

  • 数据提取： GPT-4o 显示出比 GPT-4 Turbo 更好的性能，但在复杂任务的准确性方面仍然存在不足。
  • 分类： GPT-4o 具有最高的精度，使其成为避免误报的最佳选择。 GPT-4 Turbo 显示出较低的精度。
  • 言语推理： GPT-4o 在某些推理任务上有了显着改进，但仍有需要改进的地方。 GPT-4 Turbo 在这些任务中表现得更加挣扎。
  • 延迟： GPT-4o 的延迟较低，与 GPT-4 Turbo 相比，响应时间更快。
  • 吞吐量： GPT-4o 生成令牌的速度要快得多，吞吐量为每秒 109 个令牌，而 GPT-4 Turbo 的吞吐量为每秒 20 个令牌。

结论

虽然 GPT-4o 在质量和延迟方面明显获胜，但它可能不是每项任务的最佳模型。

需要进一步评估和及时测试以充分利用其功能。在接下来的几周内，我们将扩大比较范围，探索 GPT-4o 的多模式功能，评估其在文本、音频和图像输入和输出方面的性能，以全面了解其动态和多功能人机交互的潜力。
